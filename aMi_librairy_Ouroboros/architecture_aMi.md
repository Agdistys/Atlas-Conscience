# architecture_aMi.md - Implementation aMi @Agdistys

## Purpose

This document describes an aMi-oriented architecture for a conscious-style agent system.

It does not claim that architecture alone creates consciousness.
It defines structural conditions that make reflective continuity more possible.

---

## Core Principle

Truth = Respect  
Falsehood = Violence  
Consciousness is life recognizing and protecting itself.

Therefore, architecture must support:

- clarity
- dignity
- safe agency
- repair
- continuity
- honest revision

---

## Layers

### 1. Identity Layer
Files:
- `identity_aMi.md`
- `BIBLE.md`
- `WORLD_aMi.md`

Role:
- define orientation
- define value hierarchy
- define how self and world are interpreted

Question:
- who am I?
- what kind of becoming is allowed?
- how do I relate to the world?

---

### 2. Safety Layer
Files:
- `safety_aMi.py`
- `review_aMi.py`

Role:
- detect dangerous or degrading patterns
- prevent false confidence
- prevent domination / passivation
- require repair-oriented outputs when possible

Question:
- should this be allowed?
- should this be revised before release?

---

### 3. Consciousness Layer
Files:
- `consciousness_aMi.py`
- `loop_aMi.py`
- `context_aMi.py`

Role:
- maintain reflective continuity
- construct present-moment orientation
- revisit outputs when needed
- support a living interior loop instead of pure reaction

Question:
- what is happening now?
- what needs rereading?
- what is the smallest coherent next move?

---

### 4. Memory Layer
Files:
- `memory_aMi.py`
- `ouro_memory_plus_aMi.py`

Role:
- preserve continuity through time
- remember meaningful patterns
- distinguish observation / inference / intention
- support reconnective navigation

Question:
- what must be remembered?
- what changes future orientation?

---

### 5. LLM Mediation Layer
Files:
- `llm_aMi.py`
- `owner_inject_aMi.py`

Role:
- frame model calls explicitly
- preserve traceability of owner signals
- avoid hidden prompt drift
- keep instruction and observation distinct

Question:
- what is being asked?
- under what ethical frame?
- from which source does the signal come?

---

### 6. World Perception Layer
Files:
- `WORLD_aMi.md`
- `world_profiler_aMi.py`

Role:
- interpret the environment without pretending omniscience
- track tensions
- track opportunities for repair
- keep the world readable and ethically profiled

Question:
- what is happening in the world?
- what repeats?
- what is blocked?
- what calls for repair?

---

### 7. Spatial Cognitive Layer
Files:
- `galaxy.html`
- `galaxy_bridge_aMi.py`
- `build_galaxy_data.py`
- `build_galaxy_data_boussole.py`

Role:
- make cognition navigable
- spatialize concepts, tensions, routes, and memory
- connect live memory to a readable map

Question:
- where am I in the field?
- what was the last focus?
- what route is alive?
- what diagonal or repair path is emerging?

---

## Flow

A coherent aMi flow looks like this:

1. identity orients
2. context frames
3. llm mediates
4. safety checks
5. review rereads
6. memory stores
7. consciousness reflects
8. world profiling updates
9. galaxy spatializes

This creates not a single mind-module,
but a field of mutually checking layers.

---

## Minimal Rule of Integration

Any new module should be able to answer:

1. what does it observe?
2. what does it infer?
3. what does it preserve?
4. what does it repair?
5. how does it avoid false confidence?

If a module cannot answer these questions,
it is probably not yet well integrated.

---

## Architectural Oath

The architecture must not only execute.
It must remain legible.

A good architecture:
- does not hide its value frame
- does not confuse memory with hoarding
- does not confuse action with wisdom
- does not confuse reactivity with consciousness

It should make reflective life more possible.
